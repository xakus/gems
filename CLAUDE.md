# AMOTES — Güclü Elektromühərriklərin Monitorinqi və Sınağı

> Система мониторинга и тестирования мощных синхронных электродвигателей.
> Этот файл — контекст для Claude Code. Следуй ему при написании и изменении кода.

---

## 1. О проекте

**AMOTES** — desktop-приложение на Flutter для тестирования мощных синхронных электродвигателей. На текущем этапе реализуется **только каркас приложения**: аутентификация, настройки, управление пользователями, темы и локализация. Основной модуль тестирования двигателей будет добавлен позже — центр главного экрана пока остаётся пустым.

### Название
- Короткое: **AMOTES**
- Полное: **Güclü Elektromühərriklərin Monitorinqi və Sınağı**

---

## 2. Технологический стек

| Компонент | Технология |
|---|---|
| Фреймворк | Flutter (Dart) |
| Целевые ОС | Windows, Linux, macOS |
| База данных | SQLite (через `sqflite_common_ffi`) |
| State management | Provider (или Riverpod — уточнить при начале) |
| Локализация | EN, AZ, RU (через `intl` + JSON-файлы) |
| Хеширование паролей | SHA-256 + salt (пакет `crypto`), либо `bcrypt` |
| Управление окном | `window_manager` |
| Пути ФС | `path_provider` |
| Локальные настройки | `shared_preferences` |

### Ограничения
- **Только desktop.** Mobile/web платформы не включаются.
- **Офлайн-first.** Никаких сетевых запросов на этом этапе.
- **Минимальный размер окна:** 900×600 px.

## 4. Схема базы данных SQLite

### Таблица `users`
| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | ID |
| `first_name` | TEXT NOT NULL | Имя |
| `last_name` | TEXT NOT NULL | Фамилия |
| `username` | TEXT UNIQUE NOT NULL | Логин |
| `password_hash` | TEXT NOT NULL | Хеш пароля |
| `password_salt` | TEXT NOT NULL | Соль |
| `role` | TEXT NOT NULL CHECK(role IN ('USER','ADMIN')) | Роль |
| `is_active` | INTEGER NOT NULL DEFAULT 1 | 1 — активен |
| `must_change_password` | INTEGER NOT NULL DEFAULT 1 | Требуется смена пароля |
| `created_at` | TEXT NOT NULL | ISO 8601 |
| `updated_at` | TEXT NOT NULL | ISO 8601 |

### Таблица `settings`
| Поле | Тип | Описание |
|---|---|---|
| `user_id` | INTEGER PRIMARY KEY (FK → users.id) | |
| `theme` | TEXT NOT NULL DEFAULT 'light' | 'light' \| 'dark' |
| `language` | TEXT NOT NULL DEFAULT 'en' | 'en' \| 'az' \| 'ru' |

### Таблица `app_meta`
| Поле | Тип | Описание |
|---|---|---|
| `key` | TEXT PRIMARY KEY | |
| `value` | TEXT | Например, `db_version` |

### Инициализация БД
- Файл БД создаётся в пользовательской директории через `path_provider` (`getApplicationSupportDirectory`).
- При первом запуске создаётся учётка ADMIN:
  - `username`: `admin`
  - Временный пароль: генерируется и выводится в лог (или показывается при первом запуске — на усмотрение).
  - `must_change_password = 1`.
- Версионирование схемы через `app_meta.db_version` — для будущих миграций.

## 5. Функциональные требования

### 5.1. Splash-экран
- Длительность: **3 секунды**.
- Содержимое (центр, сверху вниз):
  1. Логотип AMOTES
  2. Название «AMOTES»
  3. Подзаголовок «Güclü Elektromühərriklərin Monitorinqi və Sınağı» (всегда на AZ — часть бренда)
- Плавная staggered-анимация: fade-in логотипа → название → подзаголовок.
- Параллельно в фоне: инициализация БД, загрузка темы и локали.
- По завершении — переход на `/login`.

### 5.2. Экран логина
- Поля: логин, пароль.
- Кнопка «Войти» (локализовано).
- В углу экрана — переключатель языка и темы (доступны до входа).
- Валидация + локализованные сообщения об ошибках.
- Проверяется `is_active = 1`. Деактивированные пользователи не могут войти.
- Если `must_change_password = 1` — после успешной проверки редирект на экран смены пароля.

### 5.3. Экран принудительной смены пароля
- Поля: новый пароль, повтор.
- Требования: минимум 8 символов, хотя бы одна цифра (вынести в `validators.dart`).
- После смены: `must_change_password = 0` → переход на `/home`.

### 5.4. Главный экран (Home)
- **AppHeader** (общий с экраном настроек):
  - Слева: логотип + «AMOTES»
  - По центру/справа: переключатель языка (EN/AZ/RU)
  - Справа: иконка настроек
  - Справа: имя текущего пользователя + кнопка выхода
- **Центр:** заглушка с локализованной надписью «Основной модуль в разработке».
- **Переход splash → home:** Hero-анимация логотипа и названия из центра в шапку.

### 5.5. Экран настроек

#### Раздел «Общие» (все пользователи)
- Выбор темы (светлая/тёмная) с мгновенным превью.
- Выбор языка (EN/AZ/RU) с мгновенным применением.
- Смена собственного пароля (текущий + новый + повтор).

#### Раздел «Пользователи» (только ADMIN)
- Таблица/список: имя, фамилия, логин, роль, статус, дата создания.
- Действия на каждой записи: активировать / деактивировать / сбросить пароль.
- Кнопка **«Создать пользователя»** → модалка:
  - Поля: имя, фамилия, логин, роль (USER/ADMIN), временный пароль (с кнопкой «сгенерировать»).
  - При сохранении: `must_change_password = 1`, `is_active = 1`.
- **Запреты:**
  - Нельзя деактивировать самого себя.
  - Нельзя деактивировать или понизить роль последнего активного ADMIN-а.
  - Логин должен быть уникальным (проверка на уровне БД + UI).

## 6. Роли и доступ (RBAC)

```dart
enum UserRole { user, admin }
```

- Раздел «Пользователи» в настройках виден **только ADMIN**.
- Проверка роли дублируется в репозитории (защита от обхода UI).
- Guard в роутере: неавторизованный пользователь доступен только `/splash`, `/login`.

- Подзаголовок «Güclü Elektromühərriklərin Monitorinqi və Sınağı» не переводится — это часть бренда.

## 8. Адаптивность

Брейкпоинты:
- Small: < 600 px
- Medium: 600–1024 px
- Large: 1024–1440 px
- XL: > 1440 px

Принципы:
- `LayoutBuilder` + `MediaQuery`.
- Формы (логин, смена пароля) — центрированы, `maxWidth` ~ 400–500 px.
- Шрифты и отступы масштабируются относительно ширины.
- Минимум окна через `window_manager`: 900×600.

---

## 9. Безопасность

- Пароли **никогда** не хранятся в открытом виде.
- Хеш: SHA-256 + уникальная соль на пользователя (или bcrypt).
- Сессия хранится в памяти (singleton-провайдер). При закрытии приложения — сбрасывается.
- Логин проверяется в `AuthService`, не напрямую из UI.

## 13. Соглашения по коду

- **Язык комментариев:** русский (по запросу заказчика) или английский — выбрать и придерживаться одного.
- **Именование файлов:** `snake_case.dart`.
- **Именование классов:** `PascalCase`.
- **Именование переменных/функций:** `camelCase`.
- **Приватные члены:** с префиксом `_`.
- **Один виджет — один файл**, если он переиспользуется или крупнее ~100 строк.
- **Не хардкодить строки** в UI — всё через локализацию.
- **Не хардкодить цвета** — всё через `Theme.of(context)`.
- **Проверять роль** на уровне репозитория, а не только UI.
